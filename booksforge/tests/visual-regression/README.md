# Visual regression suite (BACKLOG §H6)

Pixel-diff Playwright tests that catch rendering drift between the
editor's HTML preview and the EPUB packager's chapter XHTML output.

## What this catches

- A change to the EPUB packager's CSS that doesn't ship in the editor
  preview stylesheet (or vice versa) → the user wrote one thing, the
  reader sees another.
- A font swap, line-height change, or scene-break ornament refactor
  that drifts only one path.
- An XHTML structural change (heading wrapper, section markers) that
  the editor preview doesn't account for.

The complementary Rust-side suite
([`crates/booksforge-export-epub/tests/visual_regression.rs`](../../crates/booksforge-export-epub/tests/visual_regression.rs))
checks the **content-level** invariant (every preview paragraph is in
the EPUB chapter body) without needing a browser; this Playwright
suite checks the **rendering** invariant (pixels match the golden).

## First-time setup

```bash
pnpm install
pnpm --filter @booksforge/visual-regression exec playwright install chromium
pnpm --filter @booksforge/visual-regression update-golden
```

Inspect the generated PNGs under `golden/` and commit them.  The
`update-golden` step also writes the per-profile fixture HTML files
under `fixtures/<profile>/` (see the **fixture generator** section
below).

## Running

```bash
pnpm --filter @booksforge/visual-regression test
```

The CI workflow (`.github/workflows/ci.yml`, job `visual-regression`,
TODO) runs the same command after `pnpm gen:fixtures` and uploads the
diff report on failure.

## Tolerance

Configured in `playwright.config.ts`:

- `maxDiffPixelRatio: 0.01` — at most 1 % of pixels may differ.
  Catches structural regressions while tolerating known
  anti-aliasing variance.
- `threshold: 0.2` — per-pixel hue/luminance delta must be ≤ 0.2 to
  count as "matching".

Tighten as the suite matures.

## Fixture generator

Fixtures live under `fixtures/<profile>/`:

- `preview.html` — the editor's HTML preview output (from
  `manuscript_to_html_chapters`) wrapped in the editor's preview CSS.
- `epub-chapter.xhtml` — extracted from a freshly built EPUB
  (chapter-001.xhtml + the EPUB's `book.css`).

Both are generated by a small Rust helper invoked via `pnpm
gen:fixtures` (TODO — slated to land alongside the first golden run).
The helper exercises the same code paths (`build_epub_bytes`,
`render_book_css`) the production export pipeline uses, so any drift
in those functions is caught here.

## Updating goldens after a deliberate change

```bash
pnpm gen:fixtures            # rebuilds fixtures/
pnpm test:update-golden      # regenerates golden/
```

Inspect the diff in your VCS and commit if the change is intentional.

## Limitations

- Headless Chromium font rendering differs from native macOS / Windows
  text layout.  This suite catches **drift between two paths in the
  same browser**, not absolute fidelity to any particular reader.
- Per-platform CI variance is constrained to the `ubuntu-22.04` Linux
  runner — the same fonts ship with the Chromium build there, so the
  goldens are stable across CI runs.  Updating goldens locally on
  macOS will produce different PNGs; either regenerate from CI or
  use a Linux container.
