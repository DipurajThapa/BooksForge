# Bundled sidecar binaries

> Tauri's `bundle.externalBin` mechanism ships these binaries
> alongside the desktop app.  They are runtime-invoked as separate
> processes (not statically linked), which is what keeps the GPL-
> family Pandoc binary licence-compatible with our own posture.
>
> **Refs:** `docs/DISTRIBUTION.md §3` (signing policy),
> `THIRD_PARTY_LICENSES.md §2.3` (licensing posture),
> `booksforge/Cargo.toml`, `tauri.conf.json` `bundle.externalBin`.

---

## What goes here

| Binary | Version | License | Purpose |
|---|---|---|---|
| `pandoc-3.5` | 3.5 | GPLv2-or-later | DOCX + PDF export sidecar |
| `epubcheck-5.1.0.jar` + `epubcheck` launcher | 5.1.0 | Apache-2.0 + W3C | EPUB validation sidecar |
| `java` (only if no system JRE) | 21 LTS | GPLv2 with Classpath Exception | EPUBCheck runtime if user does not have Java |

Each binary is referenced from `booksforge/apps/desktop/tauri.conf.json`
under `bundle.externalBin` (with the platform suffix Tauri appends:
e.g. `pandoc-3.5-x86_64-apple-darwin`).

## Per-platform layout

After running `scripts/fetch-sidecars.sh`, this directory contains:

```
binaries/
├── pandoc-3.5-aarch64-apple-darwin
├── pandoc-3.5-x86_64-apple-darwin
├── pandoc-3.5-x86_64-pc-windows-msvc.exe
├── pandoc-3.5-x86_64-unknown-linux-gnu
├── epubcheck/
│   ├── epubcheck-5.1.0.jar
│   ├── lib/   (EPUBCheck's bundled deps)
│   └── ...
├── epubcheck-runner-aarch64-apple-darwin       (small launcher script)
├── epubcheck-runner-x86_64-apple-darwin
├── epubcheck-runner-x86_64-pc-windows-msvc.exe
├── epubcheck-runner-x86_64-unknown-linux-gnu
├── CHECKSUMS.txt    (SHA-256 of every binary, regenerated by fetch-sidecars.sh)
└── README.md        (this file)
```

## Reproducibility

Run from `booksforge/`:

```bash
./scripts/fetch-sidecars.sh
```

The script:
1. Downloads each binary from its **canonical upstream source** (Pandoc
   from `github.com/jgm/pandoc/releases`, EPUBCheck from
   `github.com/w3c/epubcheck/releases`).
2. Verifies the SHA-256 against the value pinned in
   `scripts/fetch-sidecars.sh`'s `CHECKSUMS` table.
3. Places the binaries in this directory with the Tauri-required
   platform suffix.
4. Regenerates `CHECKSUMS.txt`.

**The release pipeline (`.github/workflows/release.yml`) MUST run
this script before bundling so binaries on every release tag come
from a verifiable source.**

---

## What this directory should NOT contain

- ❌ Source code of Pandoc / EPUBCheck.  We invoke them as
  processes; no source needs to ship.
- ❌ Hand-edited binaries.  If a binary is hand-edited, the
  reproducibility test in `release.yml` fails.
- ❌ Anything outside the per-platform allowlist in
  `scripts/fetch-sidecars.sh`.  Adding a new sidecar requires
  updating both the allowlist and `THIRD_PARTY_LICENSES.md §2.3`.
