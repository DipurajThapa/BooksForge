## What it is

A local-first desktop writing app. The user opens a project (a
`.booksforge/` directory bundle), writes prose in TipTap, and runs a
small fleet of "agents" that read excerpts and propose edits. Every
agent call goes to a **local** LLM (Ollama on `127.0.0.1:11434`); no
manuscript content leaves the device by default.

The app exports to DOCX, PDF, and EPUB-3 via Pandoc (DOCX/PDF) and
a custom Rust EPUB builder, with EPUBCheck verification.

## The four-layer architecture

```
┌─────────────────────────────────────────────────────────────────┐
│ L1 — Presentation                                               │
│   booksforge/apps/desktop/src-ui/                               │
│   React + TypeScript + Vite + TipTap                            │
│   booksforge/packages/{editor, preview, ui, shared-types}/      │
└────────────────────────────┬────────────────────────────────────┘
                             │ Tauri IPC (typed via ts-rs)
┌────────────────────────────▼────────────────────────────────────┐
│ L2 — App services (Tauri command handlers)                      │
│   booksforge/apps/desktop/src/commands/                         │
│   Tauri commands: typed input, typed output, tagged-union error │
└────────────────────────────┬────────────────────────────────────┘
                             │ trait calls
┌────────────────────────────▼────────────────────────────────────┐
│ L3 — Domain (pure-logic Rust, NO I/O, NO clocks)                │
│   booksforge/crates/booksforge-{domain, template, validator,    │
│       agents, prompt, memory, vocab, export}/                   │
└────────────────────────────┬────────────────────────────────────┘
                             │ trait boundaries (StorageRepository,
                             │ BundleFilesystem, OllamaClient, …)
┌────────────────────────────▼────────────────────────────────────┐
│ L4 — Infrastructure                                             │
│   booksforge/crates/booksforge-{storage, fs, ollama,            │
│       orchestrator, snapshot, export-epub, export-pandoc,       │
│       epubcheck}/                                               │
│   SQLite (via sqlx), filesystem, HTTP (loopback only),          │
│   sidecar processes.                                            │
└─────────────────────────────────────────────────────────────────┘
```

**Cross-layer rule:** L3 cannot import L4. **`cargo deny check bans`
enforces this** in CI; a build that violates the rule fails.

## Where things live (cheat sheet)

| You want to … | Touch this … |
|---|---|
| Add a new TipTap node or React panel | `booksforge/apps/desktop/src-ui/` and (for shared bits) `booksforge/packages/{editor, ui}/` |
| Add a new Tauri command | New handler in `booksforge/apps/desktop/src/commands/` + new IPC types in `booksforge/crates/booksforge-ipc/src/`. The TS bindings are auto-generated by `cargo test -p booksforge-ipc`; commit the result. |
| Add a new agent | Spec in `outputs/AGENTS.md`; prompt template in `booksforge/crates/booksforge-prompt/templates/<agent>/v1.toml`; runner wiring in `booksforge/crates/booksforge-agents/`; orchestrator dispatch in `booksforge/crates/booksforge-orchestrator/src/run.rs`. |
| Add a new SQL table or column | Migration file `booksforge/crates/booksforge-storage/migrations/00NN_*.sql`. **Forward-only** — never edit an existing migration. Then update domain types in `booksforge-domain` and queries in `booksforge-storage/src/sqlite.rs`. |
| Add a new export format | New crate (e.g. `booksforge-export-<format>`) at L4; orchestration via `booksforge-export`; Tauri command in `booksforge/apps/desktop/src/commands/export.rs`. |
| Add a new validator (KDP, EPUB, etc.) | `booksforge/crates/booksforge-validator/`; surface in pre-export gate per `outputs/EXPORT_EPUB_QA.md`. |

## The non-negotiables (the things that cause CI to scream)

1. **No `unwrap()` / `expect()` / `panic!()` in production Rust.**
   Outside `#[cfg(test)]` and `main()`, every fallible operation
   returns a typed `thiserror`-derived error. Clippy `-D warnings`
   enforces.
2. **All SQL via `sqlx::query!` / `query_as!` macros.** Compile-time
   checked against the schema in `booksforge/.sqlx/`. No string
   concatenation into SQL. No `format!` into SQL.
3. **Layer boundaries.** L3 crates cannot import L4 (`cargo deny
   check bans`).
4. **Privacy invariants.** Five rules in `outputs/SECURITY_PRIVACY.md`,
   tested by `cargo test -p booksforge-orchestrator --test
   privacy_invariants`. The most important: **no manuscript
   content ever reaches a non-loopback URL.**
5. **No GPL/AGPL/LGPL crate statically linked.** `cargo deny check
   licenses`. Pandoc and EPUBCheck are GPL/Apache; they are OK
   *because* they run as separate processes.
6. **TypeScript strict mode.** `noImplicitAny`, `strictNullChecks`,
   `noUncheckedIndexedAccess` all on.
7. **IPC types from Rust → TS via ts-rs.** `cargo test -p
   booksforge-ipc --test codegen_drift` verifies no TS file in
   `packages/shared-types/src/bindings/` is hand-written.
8. **Migrations are forward-only at runtime.** Add new SQL files;
   never edit existing ones.

## The data model (in 30 seconds)

A project is a directory bundle:

```
<project>.booksforge/
├── manifest.toml              project metadata
├── .booksforge-version        minimum compatible app version
├── project.db                 SQLite — single source of truth
├── manuscript/                Markdown mirror (projection of pm_doc, not state)
├── assets/                    images, fonts, anything the user pastes in
├── snapshots/objects/         content-addressed scene-content blobs
├── exports/                   DOCX / PDF / EPUB output
└── agent_runs/                per-run JSON dumps (audit ledger)
```

Inside `project.db`:

| Table | What it holds |
|-------|---------------|
| `nodes` | The Binder tree: chapters, scenes, folders. |
| `scene_content` | Every scene as a TipTap `pm_doc` JSON blob. |
| `entities`, `entity_aliases`, `entity_scene_appearances` | The "memory" / character bible. |
| `style_book` | Rules and preferences propagated to agents. |
| `snapshots`, `snapshot_entries` | Manual + automatic + pre-agent-edit snapshots. |
| `validator_runs`, `validator_issues` | Continuity-linter output. |
| `agent_runs`, `agent_tasks`, `agent_outputs`, `agent_applied_edits` | Audit ledger for AI work. |

Full schema in `outputs/DATA_MODEL.md`.

## The agent system (in 30 seconds)

Agents are **prompt-in / schema-out**. They cannot call tools, cannot
recurse, cannot mutate state. The orchestrator is the only mutator:
agents return *proposals*, the orchestrator validates them against
JSON schemas, and the user accepts/rejects per-edit.

Hard caps per workflow run:

- ≤ 8 LLM calls
- ≤ 10 minutes wall-clock
- ≤ 200 k tokens
- ≤ 3 retries

Every accepted edit triggers a `pre_agent_edit` snapshot so the user
can roll back. Every call is logged to the `agent_runs` /
`ai_calls` ledger.

Agent catalogue in `outputs/AGENTS.md` (11 agents in MVP,
9 V1+ agents).

## CI gates (what runs on every PR)

| Gate | What it checks |
|------|----------------|
| `cargo fmt --all -- --check` | Rust formatting |
| `cargo clippy --workspace --all-targets -- -D warnings` | Lints + the no-`unwrap` rule |
| `cargo test --workspace --all-targets` | Unit + integration tests |
| `cargo test -p booksforge-orchestrator --test privacy_invariants` | The five privacy invariants |
| `cargo test -p booksforge-export-epub --test reproducibility` | EPUB byte-for-byte reproducibility across platforms |
| `cargo test -p booksforge-ipc --test codegen_drift` | TS bindings ≡ Rust types |
| `cargo deny check licenses bans advisories sources` | License + ban + CVE supply chain |
| `pnpm typecheck && pnpm lint && pnpm test` | TS gates |
| Cold-launch p50 ≤ 1 s on `macos-14` | Performance budget |

## How to run it

```bash
# clone
git clone git@github.com:DipurajThapa/BooksForge.git
cd BooksForge/booksforge

# install JS deps (once)
pnpm install --frozen-lockfile

# Rust dev loop — start the desktop app with hot reload
cargo tauri dev

# Run all tests (Rust + TS)
cargo test --workspace
pnpm test
```

For Ollama setup, run the **Setup Wizard** the first time the app
launches — it detects Ollama, offers an install if absent, and
pulls the default model (`qwen2.5:7b-instruct-q4_K_M`).
