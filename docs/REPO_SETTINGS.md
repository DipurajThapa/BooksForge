## 1. Branch protection — `main`

Settings → Branches → Branch protection rules → `main`.

| Rule | Value | Why |
|------|-------|-----|
| Require a pull request before merging | ✅ | No direct pushes to `main`. |
| Require approvals | ✅ 1 (rising to 2 when contributor count > 3) | CODEOWNERS-required reviewers cover sensitive surfaces (`.github/CODEOWNERS`). |
| Dismiss stale approvals on new commits | ✅ | Forces re-review when the diff changes. |
| Require review from Code Owners | ✅ | Capability widening, CI changes, deny.toml edits, IPC contracts, migrations all gated. |
| Require status checks to pass | ✅ | List the checks below. |
| Require branches to be up to date before merging | ✅ | Prevents stale-merge regressions from working-tree drift. |
| Require conversation resolution before merging | ✅ | Reviewer concerns must be addressed, not ignored. |
| Require signed commits | ⏳ | Enable once contributors are on Sigstore / Gitsign. Not a blocker today. |
| Require linear history | ✅ | No merge commits — squash or rebase. |
| Include administrators | ✅ | Founder is not above branch protection. |
| Restrict who can push | ✅ Maintainers only | |
| Restrict force-pushes | ✅ Allow specific actors: none | Protects `main` history. |
| Restrict deletions | ✅ | `main` cannot be deleted. |

### Required status checks on `main`

Mirror the gating jobs in `booksforge/.github/workflows/ci.yml` and
`.github/workflows/security-scan.yml`:

- `cargo fmt`
- `cargo clippy --workspace --all-targets -- -D warnings`
- `cargo test --workspace --all-targets`
- `cargo test -p booksforge-orchestrator --test privacy_invariants`
- `cargo test -p booksforge-export-epub --test reproducibility`
- `cargo test -p booksforge-ipc --test codegen_drift`
- `cargo deny check (advisories + bans + licenses + sources)`
- `cargo audit (independent verification)`
- `pnpm typecheck`
- `pnpm lint`
- `pnpm test`

Frontend tests stay required only after audit #22 lands a meaningful
test suite; until then, keep `pnpm test` non-blocking.

### Auto-merge eligibility

Configured in `.github/workflows/dependabot-auto-merge.yml`. Trusted
buckets that auto-merge when CI is green:

- Any **patch-level** update (`version-update:semver-patch`).
- **Minor** updates to the dev-tooling allowlist: `@types/*`,
  `eslint*`, `prettier*`, `vitest`, `@testing-library/*`.

Everything else — including all minor updates outside the
allowlist, and all major updates — requires human review.

The auto-merge workflow squash-merges; the squash commit message is
auto-generated by GitHub from the PR title.

## 5. Secret scanning + push protection

Settings → Code security → Secret scanning.

| Setting | Value |
|---------|-------|
| Secret scanning | ✅ ON |
| Push protection | ✅ ON |
| Push protection bypass list | (empty) |

Push protection blocks any commit that contains a recognised secret
pattern (Apple notarisation password, AWS keys, etc.) before it
reaches the remote. **Never bypass** unless the founder has
verified the secret is a documented, intentionally-public test
fixture.

## 8. Environments + secrets

Settings → Environments.

| Environment | Purpose | Secrets present |
|-------------|---------|-----------------|
| `release-stable` | Required for `release.yml` `publish` job | `APPLE_DEVELOPER_ID_CERT`, `APPLE_DEVELOPER_ID_PASSWORD`, `APPLE_NOTARIZATION_*`, `WINDOWS_SIGNING_*`, `TAURI_SIGNING_*` |
| `release-beta` | Same secrets as stable | (same) |
| (default) | Unrestricted CI runs | none |

Each environment that holds production signing secrets must require
manual approval from a maintainer before the workflow can use it.
This protects against a supply-chain attack on the workflow file
itself accidentally leaking a signing secret.
