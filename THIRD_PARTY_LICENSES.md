# BooksForge — Third-Party Licenses & Attributions

> **Status: scaffolded.** This file enumerates the categories of bundled
> third-party material with their license posture, and provides the
> commands needed to *regenerate* the per-dependency aggregate. The
> per-crate / per-package detail must be auto-generated and committed
> as part of the release pipeline (see `MILESTONES.md` M6.E).
>

---

## 1. Licensing posture

BooksForge ships under the terms of the project [`LICENSE`](LICENSE).
Third-party material is bundled or invoked under the licenses below.
The bundling discipline is enforced by `booksforge/deny.toml`:

- **GPL-family licenses (GPL, AGPL, LGPL) are statically banned** for
  Rust crates compiled into the binary. `cargo deny check licenses`
  fails the build on any GPL crate.
- **Sidecar binaries** (Pandoc, EPUBCheck) run as separate processes
  invoked by BooksForge. They are not statically linked. Pandoc is
  GPLv2+; this is *acceptable* in this configuration because process
  invocation does not create a derivative work.

## 3. Rust crate dependencies

The aggregate license + attribution list for every Rust crate compiled
into the binary is generated by `cargo about generate` and committed.

### 3.1 Regeneration command

```bash
cargo install cargo-about
cd booksforge
cargo about init        # only on first run
cargo about generate --output-file ../THIRD_PARTY_LICENSES.rust.html about.hbs
```

> The aggregated HTML / Markdown output is checked into the repo so
> that the bundled installer can ship an offline-readable attribution
> page. The CI release job verifies the file is up to date by running
> `cargo about generate` and comparing against the committed copy.

### 3.2 Allowlisted licenses (per `booksforge/deny.toml`)

`MIT`, `Apache-2.0`, `Apache-2.0 WITH LLVM-exception`, `BSD-2-Clause`,
`BSD-3-Clause`, `ISC`, `Zlib`, `MPL-2.0`, `BSL-1.0`, `Unicode-DFS-2016`,
`CC0-1.0`, `0BSD`. Any crate not on this list fails CI.

### 3.3 Banned licenses

`GPL-2.0`, `GPL-3.0`, `AGPL-1.0`, `AGPL-3.0`, `LGPL-2.0`, `LGPL-2.1`,
`LGPL-3.0`, `SSPL-1.0`, `BUSL-1.1`. Any future ban exception must be
recorded as an ADR in `outputs/ARCHITECTURE_DECISIONS.md`.

## 4. JavaScript / TypeScript dependencies

The aggregate license list for every npm package shipped to the
desktop bundle is generated by `pnpm licenses list`.

### 4.1 Regeneration command

```bash
cd booksforge
pnpm licenses list --json > ../THIRD_PARTY_LICENSES.npm.json
pnpm licenses list --long > ../THIRD_PARTY_LICENSES.npm.txt
```

> The aggregated text is checked into the repo on every dependency
> change. As with the Rust pipeline, the release job verifies it is
> up to date.

### 4.2 Allowlisted licenses (target)

`MIT`, `Apache-2.0`, `BSD-2-Clause`, `BSD-3-Clause`, `ISC`, `0BSD`,
`Unlicense`, `CC0-1.0`. The first signed release should add a
`license-checker` step to CI to fail on anything outside this set.

```bash

( cd booksforge && pnpm licenses list --long > ../THIRD_PARTY_LICENSES.npm.txt )

# every signed release.
```

The release pipeline (see `MILESTONES.md` M6.A) MUST run the above
commands and fail if any of these files have drifted from what is
committed.
